2016年总结

发表于 2016-01-02   |   分类于 Life   |     |   阅读次数

我的2016年

2016年，我认为是一个比较失败的一年，
在这一年中，我浪费了不少机会，也失去了不少机会，

失去&不足：

  1. 在有多个选择时，有做出好的选择.(ps: 工作，生活均有)
  2. 做事情有些纠结，执行力不够，
  3. 做事情效率低,主要原因是方法，思维不正确，不专一而致。
  4. 没能坚持每周分享一篇博客，

收获:

在过去的2016年，也有一些成长，

  1. 获得了驾照
  2. 开始写原创博客，
  3. 开源了自己的第一个开源项目.
  4. 开始看书，涉及专业以及不专业的书籍(以下是2016年的书单)

    《淘宝技术这十年》
    《逻辑思维：成大事这不纠结》
    《微信帝国》

小结:

在过去的2016年里，经历了不少事情，主要从生活，技术这两方面来进行总结.
生活上：

  1. 在面对选择的时候，眼光应该往后看几年，应该更加注重以后的发展,(如:在之前的offer选择时，就因为一点薪资，浪费了一些比较好的机会，)
  2. 不与别人进行薪资上的攀比，盲目的攀比只能让自己陷于尴尬的局面
  3. 沟通能力不强，一些自己知道，理解的东西却不能通过更加清晰的思路表达出来，

技术上：

  1. 不专一，什么多想学，执行力却不够
  2. 没有看一本专业书籍，对一些基础知识掌握不扎实
  3. 技术上没有新的突破

总结: 对过去的2016年是不满意的，用几个字来概括的话就是，’纠结’，‘不专一’，‘选择错误’。

在2017年，我相信会更好，同时也会加大力度维护公众号，博客。坚持分享!

PS:祝大家2017年新年快乐!

windows系统无法安装SQLServer2005的两种解决方法

发表于 2015-12-24   |   分类于 Sqlserver   |     |   阅读次数

最近遇到童鞋谈论说,windows下重新安装SqlServer2005总是不成功,大多数是因为以前安装过，卸载后出现的问题，(以下解决方案适用于:重新安装不成功)

两种解决方法

方法一: 清理注册表

打开windows注册表，在注册表中查找一下信息，并进行删除
[-HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\MSSQLServer]
[-HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Microsoft SQL Server]
HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Search]
HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\MSSQLSERVER]
HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\MSSQLServerADHelper]
HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\SQLSERVERAGENT]
HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\MSFtpsvc]
HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\MSSCNTRS]
HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\MSSEARCH]
[-HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\MSSGatherer]
[-HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\MSSGTHRSVC]
[-HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\mssindex]
也可以下载,通过 Windows Installer Clean UP 将里面将所有关于SQL的全部删除并重启再进行安装，

方法二: 清理注册表

哈哈，你没看错，还是清理注册表，但是是不同的位置哦

在这个路径里：
HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Contro\Session Manager
点Session Manager 然后在右边有一个叫PendingFileRenameOperations的名，删之，重启电脑，再安装即可，

小结:

以上，就是重新安装sqlserver 2005失败的解决办法！希望能够帮到您!

稍等一会，会进入到完成页面，如下图所示，到此，我们已经完成了90%了，

配置jdk环境

  1. 现在我们电脑已经安装好jdk了,默认情况电脑是不能自动识别jdk的，那怎么才能让电脑识别呢，就是需要简单的配置,打开系统属性，打开方法 (计算机–>属性–>高级系统设置->高级–>环境变量)。
    如下图所示：

  2. 我们在上图中的系统属性中,新建系统变量JAVA_HOME,变量值为您的jdk安装路径，例如(D:\java\jdk\jdk1.7.0_79)。
    如下图所示:

  3. 设置完成后，我们点击确定，紧接着，我们在系统属性中，找到环境变量为path的环境变量，在变量值后面添加系统属性，(切记是添加而不是清除后添加),
    在Path环境变量值后，添加 %JAVA_HOME%/bin;%JAVA_HOME%/jre/bin;(在添加JAVA_HOME之前，如果path变量值后面没有分号，则添加分号)。
    如下图所示:

  4. Path环境变量中添加完成后，在系统属性中，添加classpath环境变量，变量值为: %JAVA_HOME%/bin;%JAVA_HOME%/jre/bin;
    如下图所示:

就这样，电脑就能识别jdk环境了？不信，那么我们来验证一下吧
